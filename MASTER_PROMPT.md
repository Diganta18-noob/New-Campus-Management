## 🔑 KNOWN PATTERNS & DECISIONS

1. **CORS** — Backend must whitelist Vercel frontend URL explicitly.
   ```js
   app.use(cors({ origin: process.env.CLIENT_URL }))
   ```

2. **MongoDB Connection** — Always wrap in try/catch with retry logic.
   ```js
   mongoose.connect(process.env.MONGO_URI)
     .then(() => console.log('MongoDB connected'))
     .catch(err => console.error(err))
   ```

3. **Axios Base URL** — Frontend uses `import.meta.env.VITE_API_URL` (Vite env syntax, NOT `process.env`).

4. **Render Cold Start** — API calls from frontend may timeout on first request. Add a loading/retry state.

5. **JWT Auth Flow** — Token stored in localStorage, sent as `Authorization: Bearer <token>` header.

## 🧠 AI SESSION RULES

When helping with this project, always:

1. **Respect the monorepo split** — frontend and backend are separate Node projects with their own `package.json`.
2. **Use Vite env syntax** in frontend (`import.meta.env.VITE_*`), never `process.env`.
3. **Never hardcode URLs** — always use env variables.
4. **Check CORS** on any new API route added.
5. **Use Mongoose models** for all DB operations — no raw MongoDB driver calls.
6. **Test locally first** before suggesting deployment changes.
7. **Treat Render as stateless** — no file writes, no in-memory session storage.

## 🐛 DEBUGGING CHECKLIST

If something breaks, check in this order:

1. Is `MONGO_URI` valid and not rotated/expired?
2. Is the Render backend awake? (ping it first)
3. Is `VITE_API_URL` pointing to the correct Render URL?
4. Is CORS blocking the request? (check browser console → Network tab)
5. Is the JWT token present and not expired?
6. Did you `npm install` after pulling latest changes?
